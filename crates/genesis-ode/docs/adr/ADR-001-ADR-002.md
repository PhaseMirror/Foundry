ADR-001: Track A — Genesis ODE Scalar Core Instantiation (Lane A)
Document ID: ADR-001
Status: ACTIVE
Version: 1.0 (Track-A only)
Date: 2026-05-19
Author: Ryan van Gelder (Phase Mirror referee synthesis) with Chris Beckingham (original scalar architecture)
Dependency: Genesis ODE v0.4.2 Technical Note
Scope: Solely the canonical scalar core engine (Lane A)
Context
The Genesis ecosystem requires a single, stable, numerically bounded reference engine for substrate-indexed persistence simulation. All Lane A engineering, Exploder stress testing, Builder assembly, tether metrics, and governance review must operate against this immutable reference without altering its semantics, ODE form, or claim boundaries.
Decision
We instantiate Track A ODE Core as the sole canonical scalar engine. It consists exactly of:

Substrate-indexed persistence state $  C_X(t)  $
The generalized ODE:$$\frac{dC_X}{dt} = \alpha(C_X^* - C_X) + \beta G_X - \gamma S_{\rm eff,X}(t) + \eta A_X(t) + \kappa \sum w_{ij}(C_j - C_i) - \delta_X$$
Impedance / drag bridge:$$\rho_X = C_X^* (1 - e^{-\lambda S_{\rm eff,X}}),\quad
\Omega_X = \frac{1}{\sqrt{1 - (\rho_X / C_X^*)^2}},\quad
D_{k,X} = \Omega_X - 1$$
Metallurgical fatigue prototype (cyclic $  S_{\rm eff,Met}(t) = S_0 \sin(\omega t)  $) as first validated substrate
Explicit validation contracts, telemetry export, and reproducible Euler (or higher-order) integration under fixed seeds
Evidence tiers restricted to [E] (externally validated), [S] (supported by simulation + calibration), [I] (internal exploratory)

Core invariants (non-negotiable):

No prime-indexed dynamics inside the scalar core (prime structure is metadata only).
Dynamics live on $  C_X(t)  $ (or future microdomain $  C_{X,i}(t)  $ only via superseding ADR).
Core may receive bug fixes, numerical improvements, and backwards-compatible telemetry — never structural mutation from experimental branches.
Every output artifact carries schema_version, run_id, source_adr_ids, tier, and provenance.

Consequences

Positive: Provides a rock-solid, replayable baseline for Exploder/Builder tether. Enables deterministic metallurgical runs and τ computation.
Negative: Requires explicit governance overhead for any future microdomain lift.
Tradeoff: Exploration lives in Lane A Exploder sandbox; core remains protected.

Related ADRs: ADR-002 (Lane A governance & admission), ADR-003 (Exploder/Builder twin), ADR-006 (test harness).

ADR-002: Track A — Lane A Governance & Module Admission Protocol (Core Instantiation Rules)
Document ID: ADR-002
Status: ACTIVE
Version: 1.0 (Track-A only)
Date: 2026-05-19
Author: Same synthesis
Dependency: ADR-001
Context
Lane A modules must remain core-compatible engineering artifacts. Any admission, perturbation harness, or builder proposal must preserve scalar-core invariants and produce auditable, provenance-carrying artifacts.
Decision
All Track A / Lane A work is governed by the following admission and operation rules:

Module Classification
Every Lane A component declares itself as such and may only import core.scalar_surface, core.drift, core.impedance in read-only mode.
Admission Criteria for Lane A Modules (Exploder, Builder, Tether, etc.)
Must satisfy ADR-001 ODE form and impedance bridge.
Must emit canonical ShrapnelMap / ResistanceCertificate / RunManifest JSON with required fields (see Phase 2 schema).
Must compute and attach tether tension τ.
Must pass architecture boundary tests (no core mutation).

Exploder (Adversarial) Rules (simulation-only)
Bounded perturbations only (amplitude_ramp, phase_shift, …).
Drift must stay ≤ ε_X per substrate or run is invalid.
Outputs shrapnel maps + resistance certificates tagged [S] or [I].

Builder (Constructive) Rules
Accepts only certified fragments with resistance certificates.
Preserves full provenance.
Emits proposals only — never mutates core or auto-promotes tiers.

Tether Policy (τ)$$\tau = \left( \frac{\text{coverage}}{\text{required\_coverage}} \right) \times \left(1 - \frac{\|\Delta_{\rm drift}\|}{\varepsilon_X} \right)$$Builder advance gated at τ ≥ 0.70 (configurable). Novelty freeze on unseen fragility classes.
Fail-Closed & Rollback
Missing tier, provenance, or certificate → reject.
Drift > ε_X or unstable shrapnel → automatic governance review + potential quarantine.
Architecture tests enforce: builder cannot import core mutation paths; exploder has no write path.


Consequences

Guarantees scalar core remains the single source of truth.
Makes Exploder/Builder a governed, auditable Lane A twin pair.
Creates clear impact signals (τ, coverage, fragility classification) for self-tuning.

Compliance Path: All code must pass tests/architecture/test_module_boundaries.py and the metallurgical integration harness before any merge.
