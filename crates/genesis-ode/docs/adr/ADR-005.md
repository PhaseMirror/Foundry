# ADR-005: Canonical Shrapnel Schema and Resistance Certificates

**Status:** PROPOSED  
**Date:** 2026-05-19  
**Author:** Gemini CLI Agent  

## Context
To ensure "fail-closed" governance, every artifact produced by the system must follow a strict, versioned schema.

## Decision
Implement strict Pydantic/dataclass schemas for:
- `ShrapnelFragment`: A single observation of a perturbation event.
- `ShrapnelMap`: A collection of fragments representing a full Exploder run.
- `ResistanceCertificate`: A signed (metadata) attestation of robustness.
- `RunManifest`: Full provenance of a simulation run.

### Mandatory Fields for all artifacts:
- `schema_version`
- `created_at`
- `run_id`
- `source_adr_ids`
- `tier` ([E], [S], or [I])
- `provenance`

## Consequences
- Facilitates automated auditing. Prevents "orphaned" or unauthenticated data from influencing the system.
