# ADR-002: Track A — Lane A Governance & Module Admission Protocol

**Status:** ACTIVE  
**Version:** 1.0 (Track-A only)  
**Date:** 2026-05-19  
**Author:** Ryan van Gelder (Phase Mirror referee synthesis) with Chris Beckingham (original scalar architecture)  
**Dependency:** ADR-001  

## Context
Lane A modules must remain core-compatible engineering artifacts. Any admission, perturbation harness, or builder proposal must preserve scalar-core invariants and produce auditable, provenance-carrying artifacts.

## Decision
All Track A / Lane A work is governed by the following admission and operation rules:

### 1. Module Classification
Every Lane A component declares itself as such and may only import `core.scalar_surface`, `core.drift`, `core.impedance` in read-only mode.

### 2. Admission Criteria for Lane A Modules (Exploder, Builder, Tether, etc.)
- Must satisfy ADR-001 ODE form and impedance bridge.
- Must emit canonical `ShrapnelMap` / `ResistanceCertificate` / `RunManifest` JSON with required fields (see Phase 2 schema).
- Must compute and attach tether tension τ.
- Must pass architecture boundary tests (no core mutation).

### 3. Exploder (Adversarial) Rules (simulation-only)
- Bounded perturbations only (`amplitude_ramp`, `phase_shift`, …).
- Drift must stay ≤ ε_X per substrate or run is invalid.
- Outputs shrapnel maps + resistance certificates tagged [S] or [I].

### 4. Builder (Constructive) Rules
- Accepts only certified fragments with resistance certificates.
- Preserves full provenance.
- Emits proposals only — never mutates core or auto-promotes tiers.

### 5. Tether Policy (τ)
$$\tau = \left( \frac{\text{coverage}}{\text{required\_coverage}} \right) \times \left(1 - \frac{\|\Delta_{\rm drift}\|}{\varepsilon_X} \right)$$
Builder advance gated at τ ≥ 0.70 (configurable). Novelty freeze on unseen fragility classes.

### 6. Fail-Closed & Rollback
- Missing tier, provenance, or certificate → reject.
- Drift > ε_X or unstable shrapnel → automatic governance review + potential quarantine.
- Architecture tests enforce: builder cannot import core mutation paths; exploder has no write path.

## Consequences
- Guarantees scalar core remains the single source of truth.
- Makes Exploder/Builder a governed, auditable Lane A twin pair.
- Creates clear impact signals (τ, coverage, fragility classification) for self-tuning.

**Compliance Path:** All code must pass `tests/architecture/test_module_boundaries.py` and the metallurgical integration harness before any merge.
