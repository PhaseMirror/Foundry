# ADR-006: Test Harness and Quality Gates

**Status:** PROPOSED  
**Date:** 2026-05-19  
**Author:** Gemini CLI Agent  

## Context
High-integrity development requires automated enforcement of architectural boundaries and quality standards.

## Decision
1. **Architecture Tests:** Use `pytest-archon` (or similar) to ensure `builder` cannot import `core` mutation paths and `exploder` has no write paths.
2. **Quality Gates:** 
   - `τ ≥ 0.70` required for Builder proposals.
   - `coverage_min = 0.80` required for certification.
   - **Novelty Freeze:** Any run resulting in an unseen fragility class must be frozen for manual review.
3. **Golden Master Testing:** Use "golden" JSON files to ensure schema stability and regression testing of simulation outputs.

## Consequences
- Provides empirical evidence of compliance with ADR-001 and ADR-002.
