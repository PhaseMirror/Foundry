# ADR-009: Automated Prime-Band Allocation for Multiplicity

**Status:** PROPOSED  
**Date:** 2026-05-20  
**Author:** Gemini CLI Agent  
**Dependency:** ADR-007, ADR-008  

## Context
As the Genesis Governance ecosystem expands across diverse substrates (Met, AI, Semi), the manual assignment of primes to features (e.g., Prime 2 for coherence, Prime 5 for logical state) becomes a scaling bottleneck and risk for interpretative drift.

## Decision
We implement a **Prime-Band Allocator** that dynamically assigns prime factors to features based on:

1.  **Feature Entropy:** High-variance features are allocated more reliable or higher-capacity prime bands.
2.  **Categorical Significance:** Threshold-driven states (ON/OFF) are assigned distinct, high-separability prime bands to maximize reconstruction scores.
3.  **Historical Feedback:** The allocator queries the `HistoryStore` to identify which prime mappings have historically produced high `reconstruction_score` values.
4.  **Quarantine Preservation:** The allocation algorithm is strictly a metadata operation and has no path to core dynamical mutation.

### Allocation Strategy:
- **Band A (Dynamics):** Primes [2, 3, 5] for continuous state vectors.
- **Band B (Structure):** Primes [7, 11] for categorical transitions and logical states.
- **Band C (Context):** Primes [13, 17, 19] for coupling weights and environmental parameters.

### Sensitivity Refinement (v0.5.0):
We implement a `sensitivity_sweep` that re-allocates prime bands if historical reconstruction scores drop below a threshold (default 0.80) or locality deltas exceed a limit (default 0.15). This ensures the encoding layer remains optimally calibrated to the current dynamical regime.

## Consequences
- **Positive:** Automates the "Deep Multiplicity" research loop. Improves reconstruction resilience across heterogeneous regimes.
- **Negative:** Introduces a dependency on historical data for optimal allocation.
- **Tradeoff:** Adaptive precision over fixed, predictable schemas.
