# ADR-020: Phase 4 – Production‑Grade Kantian Test Suite Integration

**Status**: Proposed

**Context**
- Phases 1‑3 have defined formal, algorithmic checklists for Kant’s three formulations of the Categorical Imperative (Universal‑Law, Humanity, Kingdom‑of‑Ends).
- These checklists are documented as ADR‑017, ADR‑018, and ADR‑019 respectively.
- The project already includes a `FullGovernanceTestDriver.lean` and other Lean modules that encode moral‑logic for the Genesis‑ODE system.

**Decision**
- Implement a **KantianTestSuite** module in `src/` that:
  1. Encodes the three checklists as pure‑Lean functions operating on a `Maxim` record (fields: `circumstances`, `action`, `goal`).
  2. Provides concrete instances for the “lying‑to‑the‑murderer” maxim and the “crying‑baby” maxim.
  3. Returns a `Verdict` type (`PerfectDuty`, `ImperfectDuty`, `Permitted`).
- Expose a `runKantTests` executable in `lakefile.lean` (or a new `KantTestDriver.lean`) that prints the outcomes of each formulation for each test case.
- Update `FullGovernanceTestDriver.lean` to invoke `runKantTests` after the existing governance checks, demonstrating how deontic and consequentialist modules can coexist.

**Consequences**
- Gives the codebase a **production‑ready** mechanism for applying Kantian reasoning to any proposed policy or protocol.
- Guarantees that any future ADR that introduces a new moral rule can be automatically validated against the three Kantian formulations before integration.
- Facilitates CI testing: the CI pipeline can fail if any new maxim violates a perfect duty, preserving the integrity of the moral layer.
- Aligns with the project’s overarching goal of **self‑tuning governance**: the system now formally checks both consequentialist (utilitarian) and deontological constraints.

**Next Steps**
1. Create `src/KantianTestSuite.lean` with the data structures and functions.
2. Add concrete test cases (`lyingToMurderer`, `cryingBaby`).
3. Extend `lakefile.lean` to expose `KantTestDriver`.
4. Update CI scripts to run `lake run KantTestDriver` after each build.
5. Reference this ADR from any future ADR that modifies moral logic.
