# Phase Mirror ADR Sequence Plan

This document expands on the foundational `ADR_Scaffold.md` and establishes the strategic sequence of Architecture Decision Records (ADRs) required to build out the Multiplicity Social Physics ecosystem. Each planned ADR strictly follows the formalized Lean 4 proofing + Rust implementation mandate.

## Sequence of ADR Plans

### ADR-001: Adoption of Lean 4 for Formal Verification of Architectural Decisions
*   **Context:** Architecture decisions often drift from implementation. We require a machine-checked governance model.
*   **Decision:** Use Lean 4 to formalize ADRs as dependent types, ensuring status immutability, consequence entailment, and acyclic supersession chains.
*   **Formal Proof Obligation:** Prove that `Accepted` ADRs cannot transition back to `Proposed`.
## Status
**Adopted**

### ADR-002: Sedona Spine (Rust Kernel) as the Sole Source of Truth
*   **Context:** Multiple agents and UI components risk implementing divergent logic for ESI retention, risk assessment, and social physics axioms.
*   **Decision:** Designate the Rust-based Sedona Spine as the *sole mandatory source of truth*. Agents must only transform, never compute or override, kernel decisions.
*   **Formal Proof Obligation:** Prove that all risk levels generated by the system originate from the Rust engine state.
## Status
**Adopted**

### ADR-003: WASM SDK for the Path of Integrity
*   **Context:** We need a secure, zero-drift method to expose Rust Engine logic to TypeScript UIs and LLM agents.
*   **Decision:** Enforce the Path of Integrity (`Engine → SDK → Contract → UI`) by compiling the Rust engine's read-only interfaces to WebAssembly (WASM).
*   **Formal Proof Obligation:** Prove that the WASM output is functionally isomorphic to the Rust engine's internal state.
## Status
**Adopted**

### ADR-004: Formalizing Lifebushido Triadic Scaling
*   **Context:** Social scaling (3→9→27) requires strict structural enforcement to prevent fragmentation.
*   **Decision:** Implement the triadic hierarchy as a dependent data structure in Lean 4 and a verified graph in Rust.
*   **Formal Proof Obligation:** Prove exact triadic scaling constraints and structural integrity during node transitions.
## Status
**Adopted**

### ADR-005: Multiplicity Stablecoin (MSC) Valuation Equation
*   **Context:** The economic layer requires a structural valuation mechanism tied to system coherence and embodied health.
*   **Decision:** Hardcode the valuation convergence theorem ($V_{MSC} = 1 + S + C$) into the Sedona Spine.
*   **Formal Proof Obligation:** Prove that the MSC value converges strictly to the target value (Theorem 10).
## Status
**Adopted**

### ADR-006: Phase Mirror Governance and Deployment Gates
*   **Context:** The 7-phase implementation roadmap (`MSP_3.md`) must not be bypassed or regress.
*   **Decision:** Build state-machine gates in the Sedona Spine that verify prerequisite criteria before allowing progression to the next deployment phase.
*   **Formal Proof Obligation:** Prove monotonic progression (e.g., cannot bypass Phase 2 to enter Phase 3).
## Status
**Adopted**

### ADR-007: Embodied Triad Protocols Integration
*   **Context:** Human nervous system regulation is a structural variable, not an externality.
*   **Decision:** The Rust engine will ingest embodied capacity/stress metrics to compute the system-wide embodied viability ($\mathcal{E}(r,t)$) before calculating the system state.
*   **Formal Proof Obligation:** Prove that embodied resilience enhances the overall sovereignty index (Theorem 8).
## Status
**Adopted**
