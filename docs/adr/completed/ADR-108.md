## Context (stated intent vs implementation)
The system is locked to FeMoco to avoid the 100-qudit boundary. The brief
proposes AEGISS (Atomic orbital and Entropy-based Guided Inference for Space
Selection) to semi-automatically reduce larger target molecules to a
FeMoco-compatible CAS(20,20) proxy, widening chemical coverage. Mechanisms:
orbital-entropy analysis unified with atomic-orbital projections to guide
active-space construction.

### Stated intent (documents)
- Brief "Extend Physics Simulation: Automated Active Space Expansion" — safely
  expand molecular repertoire via automated active space selection.
- `materia_commons/` and `src/f1_surface/` encode FeMoco CAS(20,20) bounds.

### Implementation reality (materia_commons/, src/)
- Active space is hard-coded to FeMoco; no general selection workflow exists.
- No orbital-entropy or AO-projection module is present.

### Manifested boundary
Leaked (unmanifested): NO — net-new capability. Critical constraint: the
generated proxy MUST stay within the 100-qudit CAS(20,20) envelope
(Constraint 2). The "safe expansion" claim is `Proposed` until this is proven.

## Decision (the lever)
Integrate a semi-automated AEGISS workflow that reduces a target molecule to a
FeMoco-compatible active space:

1. **Orbital entropy analysis**: compute per-orbital entanglement/entropy to
   rank orbital importance.
2. **Atomic orbital projection**: project candidate orbitals onto atomic
   fragments to enforce chemical meaning.
3. **Active-space construction**: combine the two to select a CAS(20,20)
   proxy; the workflow is *semi*-automated — a human ratifies the proposed
   active space before it enters a production run.
4. **Formal bound (inviolable)**: prove `ProxyWithinCAS2020` in Lean4 — every
   AEGISS-generated active space has ≤ 20 electrons / ≤ 20 orbitals (≤ 100
   qudits). Any candidate exceeding the bound is rejected before scheduling.
5. **Traceability**: the chosen active space + selection rationale is recorded
   in the WORM ledger so the proxy is reconstructible (Constraint 4).

This enables new molecular targets without violating the 100-qudit hard
boundary, preserving all mathematical guarantees.

### Objective
Integrate AEGISS (Atomic orbital and Entropy-based Guided Inference for Space
Selection) to automatically reduce large molecular targets to a CAS(20,20)
proxy, enabling expansion beyond FeMoco while respecting the 100‑qudit
boundary.

### Motivation
Clients want to simulate other transition metal complexes. Manual active space
selection is error‑prone and slow; AEGISS automates it with formal guarantees.

### Non-Goals
- Do not breach 100‑qudit limit; AEGISS must output an active space with ≤69
  qubits.
- Do not require new hardware; AEGISS runs on classical pre‑screening.

### Technical Approach
1. **Workflow**: For a new molecule, run a cheap classical DFT calculation to
   obtain orbital energies and entropy proxies. AEGISS selects the active space
   using the entropy‑based criterion.
2. **Validation**: Compare AEGISS‑reduced energies against full CASSCF on a
   small test set (e.g., Fe‑S clusters) to confirm error < 5 mHa.
3. **Integration**: AEGISS output is fed into the existing MA‑VQE pipeline;
   Q‑SQD signatures are generated as usual.

### Blockchain Anchor (Invariant 5)
- **Problem**: If AEGISS reduces a molecule to a CAS(20,20) space, the rationale
  (the DFT entropy analysis) is currently local and not externally verifiable.
- **Implementation**: The classical DFT pre-screening data justifying the
  reduction is hashed and posted to `ChemicalRationale.sol`, linked to the
  eventual FeMoco simulation.
- **Result**: The entire deductive chain — from raw DFT numbers to the reduced
  quantum circuit to the final energy — is sealed on-chain. A theoretical
  chemist in 2050 can cryptographically prove the reduction methodology was
  appropriate.

### Implementation Phases
- **Week 1‑3**: Implement AEGISS as a Python library (based on literature);
  test on known molecules.
- **Week 4‑6**: Write a formal proof (in Lean4) that AEGISS‑selected active
  space preserves chemical accuracy to within 5 mHa (requires bounding the
  truncation error); new module `ActiveSpace.lean`.
- **Week 7‑9**: Integrate with `qaas_endpoints.rs`; new endpoint
  `/simulate_with_autoreduction` accepts a molecule specification, runs AEGISS,
  then FeMoco simulation.

## Consequences
- **Positive**: wider chemical coverage; automated, chemically-meaningful
  active-space selection; 100-qudit boundary provably preserved.
- **Negative / Constraints**: semi-automated (human ratification required);
  entropy/projection heuristics may disagree — resolution policy needed;
  added pre-processing cost per new target.
- **Verification Strategy**: `ProxyWithinCAS2020` proves the bound; a test
  feeds a larger molecule and confirms the workflow emits a CAS(20,20) proxy
  and rejects any over-boundary candidate.
