# ADR-010: Prover Backend Integration & Constraint Verification

**Status:** Proposed

**Date:** 2026-06-16

**Owner:** Prover Team / Substrates

## Context
Transitioning from mock quantization to production-grade SNARK proving (using `arkworks`) introduces structural risk due to field arithmetic differences. We must bridge the gap between our current `FieldElement(u64)` mock and the backend's native field requirements without losing integrity.

## Decision: Dual-Prover Verification
We will adopt a "Dual-Prover" integration strategy to ensure cryptographic assurance.
1. **Backend**: `arkworks` (specifically using `ark-ff` and `ark-bn254`).
2. **Verification Harness**: Implement a parallel test harness that runs proofs through the existing `SafeNecessityProjector` (mock) and the new `arkworks` backend simultaneously.
3. **Equivalence Check**: A successful hookup requires an exact match between the `proof_digest` generated by the mock and the production backend.

## Consequences
- **Assurance**: Prevents silent translation errors in R1CS constraints.
- **Complexity**: Increases initial setup time due to the parallel verification harness.

## Implementation Path
1. Initialize Track E Environment with `ark-ff` and `ark-bn254` dependencies.
2. Implement `ArkworksProver` wrapper.
3. Establish the Dual-Prover test harness in `src/crypto.rs` (or equivalent).
4. Verify constraint satisfaction via parallel proof generation.
