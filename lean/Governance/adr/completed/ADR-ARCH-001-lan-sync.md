# ADR-ARCH-001 — LAN Archivum Sync: Distributed Witness Replication

**Status**: Accepted (2026-05-22)
**Deciders**: Core / Protocol

---

## Context
`witnesses.jsonl` is currently append-only and single-node. Phase 4 Step 12 introduces replication across trusted PhaseSpace nodes on a LAN. The append-only invariant and the SHA witness chain must be preserved across distributed nodes.

---

## Decision

### 1. Ordering Model: Designated Primary (Option A)
To avoid the complexity of distributed consensus research (Vector Clocks), we designate **one node as the Primary Archivum**.
- The Primary owns the canonical sequence of witnesses.
- All other nodes (Replicas) push their witnesses to the Primary.
- The Primary performs a final validation and appends to the canonical `witnesses.jsonl`.
- If the Primary is unreachable, write-unavailability is an accepted state for Phase 4.

### 2. Authority Model: Designated Primary
Maps directly to the ordering model. Replicas are not authoritative for the global ledger.

### 3. Trust Boundary: Internal-only Replication (Option X)
To prevent governance bypass via remote sandboxes:
- **Only Internal-trust witnesses replicate.**
- Witnesses generated by `External` workflows remain node-local and are never transmitted to the Primary.
- This ensures the Primary's authoritative ledger is not "laundered" with witnesses from potentially compromised remote sandboxes.

### 4. Protocol: HTTP/SSE with SAT
Replicas use the existing HTTP transport to push witnesses. Every replication request must carry a valid Signed Admission Token (SAT), verified by the Primary's ALP gate.

---

## Consequences
- **Simplicity**: No research-heavy consensus algorithm required for Phase 4.
- **Security**: Structural enforcement that untrusted scripts never reach the authoritative ledger.
- **Scalability**: Limited to LAN/VPN environments where a Primary can be easily resolved.
- **Availability**: System depends on Primary availability for global audit trail completion.

---

## Verification
- `ReplicationConfig` struct in `multiplicity-common` enforces Option X by default.
- Governance test verifies that remote `External` witnesses are rejected by the Primary's replication endpoint.
- `pscmd archivum log` shows consistent witness IDs across Primary and Replicas.
