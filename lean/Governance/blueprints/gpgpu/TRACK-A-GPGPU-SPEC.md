# Specification: Track A GPGPU Offloading Path

## 1. Overview
This document outlines the technical specification for transitioning the Goldilocks NTT and Poseidon2 Merkle tree construction from CPU SIMD (AVX2/AVX-512) to GPGPU architectures. This is necessary to overcome L3 cache saturation when trace commitments exceed 2^24 rows.

## 2. Target Architectures
The offloading path must support heterogeneous environments via unified abstractions (e.g., Vulkan or WebGPU via Rust `wgpu`), with vendor-specific optimizations for:
- **CUDA (NVIDIA):** Primary datacenter proving target.
- **Metal (Apple Silicon):** High-memory-bandwidth local proving.

## 3. Mathematical Invariants
The GPGPU implementation MUST NOT alter the Lawful Core semantics:
1. **Bit-Perfect Parity:** The GPU kernel outputs must be bit-for-bit identical to the scalar and AVX2 oracles.
2. **Goldilocks Modular Reduction:** The p = 2^64 - 2^32 + 1 arithmetic must be implemented using 32-bit limb representation to avoid 64-bit carry bottlenecks on GPUs.
3. **Witness Preservation:** The `zero_spacings` array and spectral witness data must be transported across the PCIe bus without scalarization or loss.

## 4. Pipeline Design

### 4.1 NTT Kernel (Cooley-Tukey)
- **Memory Coalescing:** Reorder butterfly operations to maximize memory coalescing.
- **Shared Memory:** Utilize GPU shared memory for local twiddle factor caching.
- **Batched Transforms:** Execute multiple independent NTTs concurrently to saturate streaming multiprocessors (SMs).

### 4.2 Poseidon2 Hashing
- **Parallel Sponges:** Execute hash absorptions in parallel across trace columns.
- **State Array:** Keep the 12-element Goldilocks state array entirely within GPU registers during the permutation rounds.

## 5. Security and Sovereign Binding
The offloaded computation must remain cryptographically bound to the Lean 4 `StabilityCertificate`.
- **πnative Witness:** The final Merkle root generated by the GPU must be hashed against the `LEAN_PROOF_HASH_108_CORE` before being written to the Lambda-Proof / Archivum substrate.
- **Fail-Closed Veto:** If the GPU context fails or a parity mismatch is detected, the runtime must fallback to AVX2 or halt.
