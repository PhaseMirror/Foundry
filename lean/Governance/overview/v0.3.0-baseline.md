# Phase Mirror v0.3.0: Technical Overview & Ecosystem Baseline

## 1. Executive Summary
Phase Mirror is a high-assurance, Rust-native agentic runtime integrated directly into the Lapce editor engine. Version 0.3.0 establishes the **Ecosystem Baseline**, a stable foundation that bridges formal mathematical proofs with production-grade operational resilience. By leveraging a "Sedona Spine" architecture, Phase Mirror ensures that all agentic actions—from code generation to legal auditing—are deterministic, auditable, and governed by a bicameral oracle system.

## 2. Core Architecture: The Sedona Spine
The fundamental principle of Phase Mirror is **Contextual Integrity**. This is achieved through the Sedona Spine, a tiered implementation strategy that prevents "semantic drift" between the mathematical model and the user interface.

*   **Engine (Rust Core)**: The sole source of truth for retention logic, risk calculation, and state transitions.
*   **SDK (WASM/TS)**: A verified projection of the core engine into the developer environment.
*   **Agentic Runtime**: A native Rust orchestrator built on Tokio and Ractor, providing isolated, event-sourced execution loops.

### Technical Guarantees:
- **Zero Drift**: UI components and external agents cannot independently calculate risk; they must transform engine-computed facts.
- **Deterministic Replay**: Append-only event logs ensure that any agent session can be replayed from the genesis event with bit-identical results.

## 3. The Bicameral Oracle Invariant
To solve the "Authority vs. Legibility" tension in AI-assisted development, Phase Mirror v0.3.0 implements a **Bicameral Governance Model**.

### Pro Chamber (The Core)
Locked by multi-signature governance and formal verification. It enforces the **L0 Invariants**:
- Contractivity checks at transpile-time.
- Spectral small-gain limits at link-time.
- Prime-typed certification for all proofs.

### Commons Chamber (The Surface)
A transparent, manifest-driven layer where operational policies and agent configurations reside.
- **Model Manifests**: YAML-driven configuration for agent prompts and tool access.
- **Heartbeat Monitoring**: Real-time observability into runtime health and session density.

## 4. Integration Paths for Partners
Phase Mirror provides three primary integration tiers for external partners and contributors:

### Tier 1: Tool Integration (The Broker)
Partners can extend the agent's capabilities by registering custom tools with the `ToolBroker`.
- **Interface**: `pm_tools::Tool` trait.
- **Isolation**: Tools run in dedicated actors, preventing crashes in one tool from affecting the runtime.

### Tier 2: Model Coordination (The Manifest)
Domain-specific experts can define new agent "personas" by updating the `models.yaml` manifest.
- **Customization**: Inject specialized prompts, required capabilities, and restricted toolsets.

### Tier 3: Editor Hooks (The Bridge)
Direct integration with the Lapce engine via the `EditorBridge`.
- **Capability**: Propose `WorkspaceEdit` objects and access editor buffers with zero-copy efficiency.
- **Safety**: All edits are subject to the same governance gates as manual code changes.

## 5. Deployment & Lifecycle
Phase Mirror v0.3.0 is distributed as a **statically linked unified binary**.
- **Cross-Platform**: Native support for Linux, macOS, and Windows (x86_64/aarch64).
- **Resilient State**: Cross-node durability via `Coerce`, allowing agent sessions to survive node failures and migrate across the network.
- **Verified CI**: Every release is gated by a headless integration suite that confirms the integrity of the agentic loop.

## 6. Conclusion: The v0.3.0 Promise
Phase Mirror v0.3.0 is not merely an AI extension; it is a **Governed Execution Environment**. It provides the first production-ready platform where the "Math-First" engineering rule is enforced by the runtime itself, ensuring that as AGI readiness advances, the system remains anchored to its constitutional values.
