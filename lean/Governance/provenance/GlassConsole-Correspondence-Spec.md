# Glass Console Provenance Specification

## Purpose
Render proof status and witness chain in human-readable format for governance audit.

## Data Sources

| Source | Path | Format |
|--------|------|--------|
| Lean Theorems | `substrates/lean/MOC/{PIRTM.lean,PWEH.lean,CompositeOperator.lean}` | Lean 4 source |
| Rust Witnesses | `state/archivum/witnesses.jsonl` | JSON Lines |
| CI Status | `.github/workflows/pirtm_ci.yml` | YAML |

## Rendered Fields

```
Proof Status
  ├─ PIRTM Theory: ✅ Theorem 2, 3 Proved (L0-1, L0-2)
  ├─ PWEH Policy: ✅ Prime Availability Enforced (L0-3)
  └─ Composite Op: ✅ Lip-contractivity (L0-1)

Witness Chain
  ├─ Length: N entries
  ├─ Last Commit: SHA
  └─ Status: Anchored to Git
```

## Integration Point
Glass Console reads `state/archivum/proof_status.json` generated by CI.
File format:
```json
{
  "lean_theorems": {
    "PIRTM": ["recursive_tensor_stability_theorem", "computational_invariance_theorem"],
    "PWEH": ["prime_five_blocked", "honest_trace_valid", "forgery_blocked"],
    "CompositeOperator": ["composite_contractive", "uniform_bounded"]
  },
  "rust_tests": "27/27 passing",
  "python_verification": "k=0.180556 < 1, Stability confirmed"
}
```
