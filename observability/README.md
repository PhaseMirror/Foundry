# Prime / Observability

This directory contains the telemetry and monitoring infrastructure for the Prime Materia Commons.

## Overview

We utilize a modern observability stack to ensure liveness, track drift rates, and monitor the health of the Sedona Spine. 

### Stack

- **Prometheus (`prometheus/`):** Time-series database configured to scrape metrics from the Rust engines, MCP servers, and policy gates. Tracks parameters such as contractivity scores, audit witness generation rates, and agent API usage.
- **Alertmanager (`alertmanager/`):** Handles alerts generated by Prometheus. Alerts are configured for critical invariant violations, such as circuit breaker trips, missed heartbeats, or unauthorized policy bypass attempts.

All metrics exported by the Prime core must be ingested here to provide a real-time dashboard of the Phase Mirror's operational stability.
